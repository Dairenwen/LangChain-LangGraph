# 练习：文档长度拆分

这是一篇专门用来练习按照文档长度进行拆分的短文。它的特点是段落比较长，主题变化不明显，适合使用 `RecursiveCharacterTextSplitter` 或 `CharacterTextSplitter` 观察 `chunk_size` 和 `chunk_overlap` 对切分结果的影响。长度拆分不关心标题层级，也不主动理解语义边界，它主要按照字符数、分隔符和重叠范围，把一段连续文本切成多个大小接近的片段。

在学校选课系统中，学生通常需要先登录教务平台，查看本学期开放的课程列表，然后根据专业培养方案、个人兴趣、时间安排和学分要求进行选择。课程信息一般包括课程名称、课程编号、任课教师、上课时间、上课地点、容量限制、已选人数、考核方式和学分数。如果一门课程人数已满，学生可能需要等待其他同学退课，或者选择同类替代课程。为了避免冲突，学生还需要检查不同课程之间的上课时间是否重叠。

选课完成后，系统通常会生成个人课表。学生应当仔细核对课表中的课程名称、周次、节次和教室信息，特别是实验课、体育课和跨校区课程。有些课程只在特定周次上课，有些课程则需要提前完成线上学习任务。若学生没有及时确认课表，可能会错过第一周的课程说明、实验安全培训或分组安排。因此，选课不是简单地点击确认，而是一个需要结合规则、时间和学习计划进行判断的过程。

如果学生在试听后发现课程难度过高、时间安排不合适，或者与其他学习任务发生冲突，可以在规定时间内申请退课。退课通常有明确截止日期，超过截止日期后，系统可能不再允许学生自行退课。部分课程由于人数、实验资源或教学安排的限制，退课还需要经过学院或任课教师审核。学生在退课之前应当确认该课程是否属于必修课，是否影响毕业要求，以及是否会导致本学期总学分低于最低要求。

长度拆分练习时，可以尝试设置不同的参数。例如，将 `chunk_size` 设置为 100、200 或 500，观察同一篇文档会被切成多少块；将 `chunk_overlap` 设置为 0、20 或 50，观察相邻片段之间是否保留了重复上下文。较小的 `chunk_size` 会让切分结果更细，但也可能破坏完整句子；较大的 `chunk_size` 能保留更多上下文，但检索时可能包含更多无关内容。合适的参数需要根据文档长度、问题类型和模型上下文窗口共同决定。
